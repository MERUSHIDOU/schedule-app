# コードレビュー

コミットされていない変更について包括的なセキュリティと品質をレビュー：

1. 変更されたファイルを取得：git diff --name-only HEAD

2. 変更されたファイルごとに以下をチェック：

**セキュリティの問題（重大）：**
- ハードコードされた認証情報、APIキー、トークン
- SQLインジェクション脆弱性
- XSS脆弱性
- 入力検証が不足している
- 安全でない依存関係
- パストラバーサルリスク

**コード品質（高）：**
- 関数 > 50行
- ファイル > 800行
- ネスト深度 > 4レベル
- エラーハンドリングが不足
- console.logステートメント
- TODO/FIXMEコメント
- パブリックAPIのJSDoc不足

**ベストプラクティス（中）：**
- ミューテーションパターン（イミュータブルを使用）
- コード/コメント内のemoji使用
- 新しいコード用テスト不足
- アクセシビリティ問題（a11y）

3. レポートを生成：
   - 重要度：CRITICAL、HIGH、MEDIUM、LOW
   - ファイル位置と行番号
   - 問題の説明
   - 提案される修正

4. CRITICALまたはHIGH問題が見つかった場合はコミットをブロック

セキュリティ脆弱性のあるコードを決して承認しない！
