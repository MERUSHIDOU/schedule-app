# Claude Code 設定

このファイルはClaude Codeがプロジェクトで作業する際の指示を定義します。

## ブランチ運用ルール

- **新しいブランチで作業すること**: 機能追加・バグ修正・ドキュメント変更など、すべての変更は新しいブランチで作業してください
- **必ずPRとして提出すること**: 変更は直接masterブランチにプッシュせず、Pull Requestとして提出してください
- ブランチ命名規則:
  - 機能追加: `feat/<機能名>`
  - バグ修正: `fix/<修正内容>`
  - ドキュメント: `docs/<内容>`
  - リファクタリング: `refactor/<内容>`

## ドキュメントルール

### TODO.md
- **読み取り**: 許可
- **編集**: **禁止**（ユーザーの許可が必要）
- 今後の実装予定・修正予定を記載したドキュメント
- 編集が必要な場合は、必ずユーザーに確認を取ること

## プロジェクト概要

- PWA対応のスケジュール管理アプリ
- React + TypeScript + Vite
- GitHub Pagesでホスティング

## 開発コマンド

```bash
npm run dev      # 開発サーバー起動
npm run build    # プロダクションビルド
npm run preview  # ビルド結果プレビュー
```
